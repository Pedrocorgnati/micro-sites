14:T21de,<h1>Como Migrar do WordPress para Next.js Sem Perder SEO</h1>
<p><strong>Resposta direta:</strong> é possível migrar WordPress para Next.js sem perder posicionamento no Google, desde que você mantenha as mesmas URLs (ou implemente redirects 301 corretos), preserva todos os metadados e recria o sitemap corretamente. O processo leva de 2 a 6 semanas dependendo do volume de conteúdo.</p>
<hr>
<h2>Por Que Migrar do WordPress para Next.js?</h2>
<p>Os motivos mais comuns que levam empresas a migrar:</p>
<ul>
<li><strong>PageSpeed mobile abaixo de 65</strong> mesmo após otimizações no WordPress</li>
<li><strong>Custo de hosting gerenciado premium</strong> (Kinsta, WP Engine) acima de R$ 500/mês</li>
<li><strong>Vulnerabilidades de segurança</strong> recorrentes via plugins desatualizados</li>
<li><strong>Integrações bloqueadas</strong> que não encontram plugin adequado</li>
<li><strong>Dependência técnica</strong> de um único desenvolvedor WordPress</li>
</ul>
<hr>
<h2>O Que Pode Dar Errado (e Como Evitar)</h2>
<p>Antes do passo a passo, os erros que destroem posicionamento:</p>
<h3>1. URLs Mudaram Sem Redirect 301</h3>
<p><strong>O problema:</strong> o Google indexou <code>seusite.com/blog/meu-artigo/</code>. Se no Next.js a URL virar <code>seusite.com/blog/meu-artigo</code> (sem barra final), ou <code>seusite.com/artigos/meu-artigo</code>, o Google trata como nova página e a antiga some.</p>
<p><strong>A solução:</strong> mapeie cada URL antiga e implemente redirect 301 no <code>next.config.js</code> ou no servidor. Mantenha os redirects por pelo menos 12 meses.</p>
<h3>2. Metadados Perdidos</h3>
<p>Títulos, meta descriptions e Open Graph que existiam via YOAST precisam ser recriados em cada página Next.js usando <code>generateMetadata</code>.</p>
<h3>3. Imagens Sem Alt Text</h3>
<p>O WordPress pode ter imagens sem alt text. Na migração, é o momento de corrigir — mas manter os alts existentes para não criar mudanças desnecessárias.</p>
<h3>4. Sitemap Incorreto</h3>
<p>O WordPress gera sitemap automático. O Next.js precisa que você implemente (manualmente ou via <code>next-sitemap</code>). Um sitemap incorreto atrasa a re-indexação em semanas.</p>
<hr>
<h2>Passo a Passo da Migração</h2>
<h3>Fase 1: Auditoria Pre-Migração (3–5 dias)</h3>
<p><strong>1.1. Exportar inventário completo do WordPress</strong></p>
<pre><code class="language-bash"># Via WP-CLI
wp post list --post_type=post --post_status=publish --format=csv &gt; posts.csv
wp post list --post_type=page --post_status=publish --format=csv &gt; pages.csv
</code></pre>
<p>Documente: total de posts, páginas, categorias, tags, imagens em <code>/wp-content/uploads/</code>.</p>
<p><strong>1.2. Mapear todas as URLs indexadas</strong></p>
<p>Use Google Search Console → Performance → Páginas para listar todas as URLs indexadas pelo Google. Exporte para CSV. Essas são as URLs que PRECISAM continuar funcionando.</p>
<p><strong>1.3. Registrar backlinks de alta autoridade</strong></p>
<p>Use Ahrefs ou Semrush para exportar os 50 backlinks mais importantes. As URLs de destino desses links são prioritárias para preservar.</p>
<p><strong>1.4. Baseline de métricas</strong></p>
<p>Documente antes da migração:</p>
<ul>
<li>Número de palavras-chave ranqueando (Search Console)</li>
<li>Tráfego orgânico mensal</li>
<li>Posição média das 20 principais palavras-chave</li>
</ul>
<h3>Fase 2: Exportação do Conteúdo WordPress (1–2 dias)</h3>
<p><strong>2.1. Exportar via REST API</strong></p>
<pre><code class="language-javascript">// Exportar todos os posts em JSON
const response = await fetch(&#39;https://seuwordpress.com/wp-json/wp/v2/posts?per_page=100&amp;_fields=id,slug,title,content,excerpt,date,categories,tags,yoast_head_json&#39;);
const posts = await response.json();
</code></pre>
<p><strong>2.2. Baixar todas as imagens</strong></p>
<pre><code class="language-bash"># Sincronizar uploads do WordPress localmente
rsync -avz usuario@servidor:/var/www/html/wp-content/uploads/ ./public/uploads/
</code></pre>
<p><strong>2.3. Exportar categorias e tags</strong></p>
<pre><code class="language-javascript">const categories = await fetch(&#39;https://seuwordpress.com/wp-json/wp/v2/categories?per_page=100&#39;).then(r =&gt; r.json());
const tags = await fetch(&#39;https://seuwordpress.com/wp-json/wp/v2/tags?per_page=100&#39;).then(r =&gt; r.json());
</code></pre>
<h3>Fase 3: Estrutura Next.js (3–7 dias)</h3>
<p><strong>3.1. Preservar estrutura de URLs</strong></p>
<p>Se o WordPress usava <code>/blog/{slug}/</code>, configure o Next.js da mesma forma:</p>
<pre><code>app/
  blog/
    [slug]/
      page.tsx
</code></pre>
<p>Se precisar mudar a estrutura, implemente redirects em <code>next.config.js</code>:</p>
<pre><code class="language-javascript">// next.config.js
module.exports = {
  async redirects() {
    return [
      {
        source: &#39;/antigo-caminho/:slug&#39;,
        destination: &#39;/novo-caminho/:slug&#39;,
        permanent: true, // 301
      },
    ];
  },
};
</code></pre>
<p><strong>3.2. Recriar metadados com generateMetadata</strong></p>
<pre><code class="language-typescript">// app/blog/[slug]/page.tsx
export async function generateMetadata({ params }: Props): Promise&lt;Metadata&gt; {
  const post = await getPost(params.slug);
  
  return {
    title: post.yoast_title || post.title.rendered,
    description: post.yoast_description || post.excerpt.rendered.replace(/&lt;[^&gt;]+&gt;/g, &#39;&#39;),
    openGraph: {
      title: post.yoast_og_title,
      description: post.yoast_og_description,
      images: [post.featured_image_url],
    },
  };
}
</code></pre>
<p><strong>3.3. Implementar Schema.org</strong></p>
<pre><code class="language-typescript">// Dados estruturados para artigo de blog
const articleSchema = {
  &#39;@context&#39;: &#39;https://schema.org&#39;,
  &#39;@type&#39;: &#39;Article&#39;,
  headline: post.title,
  datePublished: post.date,
  author: { &#39;@type&#39;: &#39;Person&#39;, name: post.author_name },
  publisher: {
    &#39;@type&#39;: &#39;Organization&#39;,
    name: &#39;SystemForge Digital&#39;,
    logo: { &#39;@type&#39;: &#39;ImageObject&#39;, url: &#39;https://seusite.com/logo.png&#39; },
  },
};
</code></pre>
<h3>Fase 4: Sitemap e Robots (1 dia)</h3>
<p><strong>4.1. Instalar next-sitemap</strong></p>
<pre><code class="language-bash">npm install next-sitemap
</code></pre>
<pre><code class="language-javascript">// next-sitemap.config.js
module.exports = {
  siteUrl: &#39;https://seusite.com.br&#39;,
  generateRobotsTxt: true,
  changefreq: &#39;weekly&#39;,
  priority: 0.7,
  sitemapSize: 5000,
};
</code></pre>
<p><strong>4.2. Submeter sitemap no Search Console</strong></p>
<p>Após o lançamento, acesse Search Console → Sitemaps e submeta <code>https://seusite.com.br/sitemap.xml</code>.</p>
<h3>Fase 5: Lançamento (1–2 dias)</h3>
<p><strong>Checklist pré-lançamento:</strong></p>
<ul>
<li><input disabled="" type="checkbox"> Todas as URLs do inventário testadas e respondendo 200</li>
<li><input disabled="" type="checkbox"> Todos os redirects 301 testados</li>
<li><input disabled="" type="checkbox"> Sitemap.xml acessível e válido</li>
<li><input disabled="" type="checkbox"> Robots.txt não bloqueando crawlers</li>
<li><input disabled="" type="checkbox"> SSL ativo (HTTPS) e certificado válido</li>
<li><input disabled="" type="checkbox"> Google Analytics/GTM configurado</li>
<li><input disabled="" type="checkbox"> Search Console verificado no novo site</li>
</ul>
<p><strong>Lançamento gradual recomendado:</strong></p>
<p>Se o site tem alto tráfego, considere lançar primeiro em staging com <code>.htpasswd</code> para auditoria interna antes de apontar o DNS.</p>
<h3>Fase 6: Monitoramento Pós-Migração (30–90 dias)</h3>
<p><strong>Semana 1:</strong> monitore Search Console diariamente. Erros 404 aparecerão — corrija imediatamente.
<strong>Semana 2–4:</strong> acompanhe impressões e cliques. É normal uma queda leve seguida de recuperação.
<strong>Mês 2–3:</strong> as métricas devem igualar ou superar o período pré-migração.</p>
<hr>
<h2>Resultado Esperado</h2>
<p>Com migração bem executada:</p>
<ul>
<li><strong>PageSpeed mobile:</strong> 45–65 (WordPress) → 90–99 (Next.js)</li>
<li><strong>Tráfego orgânico:</strong> manutenção nos primeiros 60 dias, crescimento a partir do mês 3</li>
<li><strong>Custo operacional:</strong> redução de 60–80% (Vercel Free vs hosting gerenciado)</li>
</ul>
<p><strong>Fazemos migrações WordPress → Next.js com garantia de posicionamento.</strong> Entre em contato para avaliação do seu site.</p>
